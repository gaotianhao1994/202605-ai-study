## 步骤1：理解思考引擎

### 1.1 什么是思考引擎？

思考引擎是框架的核心，负责处理输入问题并生成决策。它模拟了人类的思考流程：

```
输入问题 → 感知理解 → 分析问题 → 生成方案 → 评估方案 → 做出决策
```

### 1.2 核心组件

打开 `thinking_engine.py`，您会看到两个主要类：

**Thought 类** - 单个思考单元：
```python
class Thought:
    def __init__(self, content, thought_type, confidence):
        self.content = content      # 思考内容
        self.thought_type = thought_type  # 思考类型（观察、分析、决策等）
        self.confidence = confidence      # 置信度（0.0-1.0）
```

**ThinkingEngine 类** - 核心思考引擎：
```python
class ThinkingEngine:
    def think(self, input_problem):
        # 执行完整思考流程
        self._perceive(input_problem)   # 感知阶段
        self._analyze(input_problem)    # 分析阶段
        self._generate_solutions()      # 生成方案
        self._evaluate_solutions()      # 评估方案
        self._decide()                  # 做出决策
```

## 步骤2：理解自我反思引擎

### 2.1 什么是自我反思？

自我反思是对思考过程的评估和改进。它检查：
- 思考过程是否完整
- 逻辑是否一致
- 证据是否充分
- 是否存在认知偏差

### 2.2 核心组件

打开 `self_reflection.py`，主要类是 `SelfReflectionEngine`：

```python
class SelfReflectionEngine:
    def reflect(self, thought_history, outcome=None):
        # 评估完整性
        self._assess_completeness(reflection)
        
        # 评估一致性
        self._assess_consistency(reflection)
        
        # 评估证据充分性
        self._assess_evidence(reflection)
        
        # 识别认知偏差
        self._identify_biases(reflection)
        
        # 生成改进建议
        self._generate_suggestions(reflection)
```

### 2.3 认知偏差识别

框架可以识别以下常见偏差：
- **确认偏差**：只寻找支持自己观点的证据
- **过度自信**：对自己的判断过于自信
- **锚定效应**：过度依赖初始信息
- **从众效应**：跟随他人意见

### 3.3 记忆类型

- **fact**：事实知识（如"水的沸点是100°C"）
- **experience**：经验（如"上次用这个方法成功了"）

## 步骤6：理解核心概念

### 6.1 思考流程

```
输入 → 感知 → 分析 → 生成 → 评估 → 决策
  ↓
反思 ←───────┘
  ↓
学习（存储经验）
```

### 6.2 为什么需要自我反思？

- **提高决策质量**：通过评估思考过程发现不足
- **避免认知偏差**：识别并纠正思维误区
- **持续学习**：从经验中吸取教训

### 6.3 记忆系统的作用

- **提供背景知识**：帮助思考引擎做出更准确的决策
- **存储经验**：记录成功和失败的案例
- **支持学习**：通过记忆巩固实现长期学习
